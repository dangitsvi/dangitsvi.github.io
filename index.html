Descent Campaign Tracker

Android Application Development

Google Play Store Listing

This app makes tracking Descent 2e campaigns easier by allowing users enter campaign information with minimal typing required. Information is entered on fragments seperated by player, which updates a summary page that the user can use for easy reference

This app was the first programming project I've ever worked on, and it was working on this project that inspired me to become a developer. I had the idea for this app when playing Descent with multiple groups of friends. Each playthrough takes roughly 20-30 hours split over multiple sessions. Keeping record of everything became very tedious, and so my brother suggested that I try making an app.

I took on this project thinking that the app was simple enough to finish in a couple weeks, but it evolved into a couple of months. I found myself constantly looking for ways to make the app better. I pored over java tutorials and user interface guides so I could increase the user experience. I was completely addicted to the process. This app was what inspired me to pursue programming as a career.

Event Funder

API Construction

Event Funder Live Demo

EventFunder is a web application that allows users to post an event with an approximate total cost. The host can then send a link out to the event attendees where they will commit to attending the event along with committing to paying their share of the cost. The goal of EventFunder is to alleviate the financial burden of hosting an event with friends and family.

Event Funder was built on a team of three in four days. The team included two back-end developers, and one front-end. My role in this project was to build the backend for the website. I mananged to finish the minimal viable product version of the API in two days. This included learning and incorporating backend authentication with Bcrypt and Json Web tokens, building the API routes, and creating database models with MongoDB. My next task was to incorporate a payment service like paypal, but our front-end dev was stuggling with Angular, so I jumped on to the front-end to help him out. I had no angular experience before hand, but I managed to learn enough to help with the front-end authentication, routing, and general bug fixes regarding scoping.

Unfortunately, due to our inexperience with angular at the time, we did not get all of the functionality hooked up in time, and our front-end dev only had a few hours to work on CSS styling. Although the front-end is mostly unfinished, I learned a lot about working with a Front-end developer and managing client to server communication.

Crave

Full-Stack JavaScript Development

Crave Live Demo

Crave is a web application where users can search for the best place to get the dish that they are craving. This information is generated by the website's users and is stored in a MongoLab data base. We leveraged the Yelp API to allow users to search for the restaurant when posting in order to increase the ease of use and accuracy of restaurant information.

Crave was built on a team of three in four days. The team was comprised of three Full-Stack JavaScript developers, so every member had a hand in building the front and back-ends.We spent only a day putting our API together. We used Node.js with Express to build the server, and used MongoDB for our database. My role in building the API was to build the routes, design the MongoDB Schema, integrate Yelp API, and write unit tests.

For the front-end, we used Angular to build everything. My role in building the front end was building templates and directives for the homepage route and food posting route, constructing reusable yelp card template for displaying yelp information in multiple areas, building controllers to handle Ajax request and other page functionality. I also assisted with miscellaneous CSS styling
